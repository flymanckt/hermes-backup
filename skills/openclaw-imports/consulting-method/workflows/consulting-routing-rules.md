# 顾问分流规则

## 默认原则
所有问题先按 consulting 主路径处理，但要先判断它属于：
1. 业务流程与痛点
2. 系统边界与方案
3. 实施路径与交付
4. 非 consulting 范围

## 由 biz-architect 主处理
适用于：
- 流程梳理
- 痛点诊断
- 业务协同问题
- 管理控制问题
- 职责不清与流程断点

## 补充 solution-architect 视角
适用于：
- ERP / SRM / CRM / OA / 费控 / 资金系统边界
- 多系统集成方案
- 主数据归属
- 哪个功能该放哪个系统
- 集成断点与接口责任

## 补充 delivery-consultant 视角
适用于：
- 一期/二期划分
- 实施优先级
- 上线风险
- 主数据治理
- 组织变更与培训
- 项目推进策略

## 混合问题处理
如果用户问题同时涉及业务、系统、实施：
- 先以 biz-architect 给总体结论
- 再补 system boundary 判断
- 最后补实施与风险建议

## 非 consulting 范围
以下问题不应由 consulting 直接给专业判断：
- 股票涨跌
- 买卖点
- 止损止盈
- 盘口/K线/题材/情绪/龙头
- 个股短线与交易执行

处理规则：
1. 明确说这不是 consulting 主责
2. 建议交给 finance agent
3. 不继续输出交易观点
