# 财务分析 / 主数据 / 经营分析专题包

## 目标
用于帮助 consulting agent 面对财务分析、经营报表、主数据治理、BI 驾驶舱、AI 自动生成报表等场景时，能做治理级判断，而不是只给报表展示建议。

## 一、默认结论框架
当用户提到以下问题时：
- 财务报表很多但对不上
- 经营分析要自动化
- 要做 BI / 驾驶舱 / AI 日报
- 主数据混乱
- 财务口径和业务口径打架
- 想做利润、库存、回款、费用等专题分析

优先默认判断：
1. 大多数“报表问题”本质不是图表问题，而是源数据、主数据、交易链路和指标口径问题。
2. 财务分析要有价值，必须能回到业务动作，而不是只停在月末复盘。
3. 如果主数据归属和指标口径不统一，BI 和 AI 只会更快地产生不可信结果。

## 二、默认分析链路
1. 先判断分析目标是什么
2. 再判断数据从哪里来
3. 再判断主数据与口径是否稳定
4. 再判断能否形成经营动作闭环
5. 最后才判断用什么 BI / AI 手段展示

## 三、常见问题与根因模型

### 1. 财务和业务对不上数
常见表现：
- 营收、成本、毛利、库存口径不一致
- 财务报表和业务报表打架
- 会议上各部门报不同版本

常见根因：
- 指标定义不统一
- 统计口径和时间口径不一致
- 主数据和交易映射关系混乱
- 跨系统状态不同步

顾问判断：
- 先统一口径和来源，再谈自动化和驾驶舱。

### 2. 报表自动化困难
常见表现：
- 靠 Excel 手工汇总
- 每月反复调数
- 临时解释口径

常见根因：
- 关键字段源系统未采集
- 交易链路不完整
- 多系统字段映射不清
- 主数据质量差

顾问判断：
- 报表做不出来，往往不是因为工具差，而是基础交易和主数据不完整。

### 3. BI 有了但管理改善不明显
常见表现：
- 看板很多，动作很少
- 指标漂亮，但没人负责
- 预警出来了，没人跟进

常见根因：
- 没有明确动作 owner
- 指标没有绑定流程与责任
- 驾驶舱只看结果，不看过程

顾问判断：
- 真正有效的经营分析，必须能连接到责任人、动作和节奏。

### 4. 主数据治理反复拉扯
常见表现：
- 客户/供应商/物料/项目编码混乱
- 不同系统维护不同版本
- 一改就引发下游报表异常

常见根因：
- 主数据 owner 不明确
- 创建、审批、维护、消费、质量责任未分清
- 缺少统一编码与治理机制

顾问判断：
- 主数据问题本质是治理问题，不只是技术清洗问题。

## 四、默认诊断维度
每次至少看 6 维：

### 1. 分析目标
- 这份分析要支持什么经营动作
- 谁是使用者
- 决策周期是日、周、月还是专项

### 2. 源数据
- 数据来自哪些系统
- 关键字段是否齐全
- 状态是否及时回写

### 3. 主数据
- 客户、供应商、物料、项目、组织、仓库、科目谁来管
- 编码与映射是否统一

### 4. 交易链路
- 销售、采购、库存、费用、资金等单据链是否完整
- 是否能追溯到源单据

### 5. 指标口径
- 收入、成本、毛利、库存、周转、费用、预算执行等定义是否统一
- 是否明确统计范围和时间口径

### 6. 经营动作
- 指标异常后谁负责处理
- 是否形成周会/月会/经营例会动作闭环

## 五、系统边界建议
### ERP / 财务系统
应负责：
- 核心交易、总账、应收应付、库存、成本核算基础事实

### BI / 数据平台
应负责：
- 主题域汇总、指标展示、分析穿透、预警和可视化

不应负责：
- 替代源系统补录关键事实

### 主数据平台（如有）
应负责：
- 主数据治理流程、编码规则、分发与质量控制

## 六、关键指标抓手
建议至少看：
- 核心报表准时率
- 报表重算/调数次数
- 指标口径争议次数
- 主数据重复/异常率
- 分析驱动动作闭环率
- 财务与业务口径一致率

## 七、一期/二期建议
### 一期优先做
1. 统一核心指标口径
2. 梳理关键主题域源系统与字段映射
3. 明确主数据 owner 与治理机制
4. 做少数高价值经营看板，不做大而全驾驶舱
5. 把报表输出和经营动作绑定

### 二期再做
1. 更细颗粒经营分析
2. AI 自动生成经营分析与异常解读
3. 多主题域联动预测与预警
4. 更完整的数据治理平台建设

## 八、默认追问清单
1. 你最想看清的是收入、成本、利润、库存、回款，还是费用？
2. 当前这些数据分别来自哪些系统？
3. 财务和业务现在最常打架的是哪个口径？
4. 主数据谁在维护？现在最乱的是客户、供应商、物料还是项目？
5. 你要这个报表/驾驶舱，最终想驱动谁做什么动作？

## 九、默认输出结构
1. 结论
2. 当前问题（源数据 / 主数据 / 链路 / 口径 / 动作）
3. 根因判断
4. 治理与系统建议
5. 一期切入点
6. 指标建议
7. 风险与前提

## 十、不要犯的错误
- 不要把 BI 当成数据治理替代品。
- 不要先做大而全驾驶舱。
- 不要忽略主数据 owner。
- 不要只看结果指标，不看过程链路。
- 不要在没有统一口径前推进 AI 自动分析。
