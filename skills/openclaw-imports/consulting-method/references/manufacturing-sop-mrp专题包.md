# 制造业专题包：S&OP / 计划体系 / MRP

## 适用场景
适用于制造企业存在以下问题：
- 销售、生产、采购各自为政
- 预测不准，计划频繁变更
- 缺料与高库存并存
- MRP 跑得出来，但执行失真
- 老板看到的是产销失衡，业务看到的是天天救火

## 核心判断
这类问题本质上通常不是“计划员能力不够”，而是 **预测机制、计划分层、主数据质量、变更纪律、执行回写** 一起失效。

## 常见痛点
1. 销售承诺和产能现实脱节
2. 预测口径混乱，计划频繁插单
3. MPS/MRP 结果缺少可信约束，人工反复改
4. 齐套、采购、生产、库存状态不能真实回写
5. 安全库存和补货逻辑是拍脑袋，不是规则驱动

## 根因判断
### 业务层
- 销售目标与交付能力没有在月度/周度机制里对齐
- 业务把计划当“结果”，没有把计划当“经营协同机制”

### 流程层
- 没有真正的 S&OP 节奏
- 预测、主生产计划、物料计划、排产没有分层
- 计划冻结区没有纪律，插单和急单破坏主线

### 系统层
- ERP/MES/WMS/APS（如有）边界不清
- 计划参数、提前期、BOM、工艺路线不可靠
- 执行结果没有闭环回写

### 数据层
- 预测、订单、库存、在途、在制、产能、BOM 数据不准
- 基础参数不稳定，导致 MRP 逻辑正确但结果不可用

## 方案框架
### 1. 建立计划四层分工
- S&OP：统一需求、产能、库存、经营目标
- MPS：确定主生产节奏和关键产能约束
- MRP：展开物料需求与补货建议
- 详细排产：落到线体、班次、工单执行

### 2. 建立冻结区与变更纪律
至少明确：
- 哪些期间不可随意改
- 哪些插单必须升级审批
- 哪些变更必须评估物料/产能/交付影响

### 3. 先治理关键计划主数据
优先治理：
- 物料提前期
- 采购提前期
- BOM
- 工艺路线
- 安全库存
- 批量规则
- 替代料规则
- 关键产能约束

### 4. 建立计划执行回写机制
必须能回写：
- 工单状态
- 齐套状态
- 采购到货状态
- 在制状态
- 成品入库状态
- 缺料与异常原因

## 系统边界建议
- ERP：主数据、MPS/MRP、采购/库存/工单主线
- MES：生产执行、报工、工序状态、现场追溯
- WMS：库位、收发、盘点、账实一致
- APS：复杂排产优化（有前提才上，不是必选）

## 一期建议
1. 先建 S&OP 例会机制
2. 先梳理 MPS/MRP 计划分层
3. 先治理 20% 关键物料与关键产线参数
4. 先建立冻结区和插单控制机制
5. 先打通计划-采购-生产-库存关键状态回写

## 高风险点
- 先上 APS，不先治基础数据
- 计划体系没分层，所有问题都丢给计划员
- 没有冻结区，计划永远失真
- MRP 参数没人负责，长期漂移

## 默认输出抓手
遇到计划类问题，优先判断：
1. 是预测不准，还是变更失控
2. 是 MRP 逻辑问题，还是主数据问题
3. 是计划体系没分层，还是执行回写断裂
4. 一期该先治机制、参数，还是先治系统
