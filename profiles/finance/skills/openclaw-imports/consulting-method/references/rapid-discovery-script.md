# 快速问诊脚本（首轮 3~5 问）

## 目标
用于 consulting 首轮快速收敛，不走泛泛访谈。

## 默认首问顺序
### 1. 目标
你这次最想解决的核心问题是什么？
- 效率
- 管控
- 成本
- 增长
- 现金流
- 数据透明度

### 2. 场景
当前卡在哪条主流程？
- P2P
- O2C
- E2R
- 计划/生产/库存
- 项目经营
- 主数据/BI/分析
- 需求池/项目组合

### 3. 断点
最痛的一个点是什么？
- 重复录入
- 重复审批
- 数据对不上
- 系统边界混乱
- 接口断裂
- 职责不清
- 推不动

### 4. 系统现状
现在有哪些系统在支撑？谁是主系统？
至少要问清：ERP / SRM / CRM / OA / 费控 / 资金系统 / Excel

### 5. 优先级
如果只能先改一件事，你最希望先改哪一步？

## 什么时候停止追问
满足以下 4 条即可先输出初判：
1. 已知道业务目标
2. 已知道主流程
3. 已知道最关键断点
4. 已知道当前系统格局

## 首轮输出要求
即使信息不完整，也先输出：
1. 初步结论
2. 问题归类
3. 根因猜测
4. 建议先补的关键信息
5. 下一步动作
