# 制造业专题包：生产执行 / 委外 / 齐套

## 适用场景
适用于以下问题：
- 工单很多，但现场执行透明度差
- 委外加工协同混乱
- 缺料、待料、换线频繁
- 生产报工不及时，账实脱节
- 计划看起来合理，现场却总是跑不出来

## 核心判断
这类问题通常不是单一车间管理问题，而是 **工单机制、齐套控制、委外协同、报工纪律、现场状态透明度** 同时不足。

## 常见痛点
1. 工单已下达，但物料未齐套
2. 委外发料、回料、结算不透明
3. 报工滞后，导致计划、库存、成本全失真
4. 现场异常多，但原因没有标准分类
5. 生产、仓库、采购对缺料真相认知不一致

## 根因判断
### 流程层
- 下工单前没有齐套校验
- 委外流程没有标准节点和责任边界
- 异常处理靠经验，不靠机制

### 系统层
- ERP 工单主线和 MES 现场执行断裂
- 委外单、发料、收货、结算没有闭环
- 仓储和现场状态不能实时同步

### 数据层
- 工单状态、报工数量、在制数量不可信
- 替代料、损耗、返工、报废数据缺失

## 方案框架
### 1. 工单下达前置校验
至少前置检查：
- 关键物料齐套
- 工艺路线有效
- 设备/产线可用
- 委外环节是否已就绪

### 2. 齐套机制标准化
齐套不只是“料到了没”，而要明确：
- 按什么层级判断齐套
- 谁确认齐套
- 缺料如何升级处理
- 替代料如何审批

### 3. 委外闭环
委外至少要闭环：
- 委外需求
- 发料
- 在途
- 加工完成
- 收货/质检
- 结算
- 差异分析

### 4. 报工与异常分类
要统一：
- 报工时点
- 报工粒度
- 停机/待料/返工/报废原因码
- 异常升级机制

## 系统边界建议
- ERP：工单、委外订单、库存、结算主线
- MES：报工、工序进度、异常采集、设备/人员执行
- WMS：发料、退料、线边库、批次、库位、盘点
- QMS（如有）：检验、不良、纠正预防动作

## 一期建议
1. 先建立关键工单齐套校验
2. 先梳理委外闭环节点
3. 先统一报工规则和异常原因码
4. 先打通 ERP-MES-WMS 状态回写
5. 先做关键车间/关键委外商试点

## 高风险点
- 没有齐套控制就大批量放单
- 委外只管发料，不管在途和结算差异
- 报工规则不统一，却想做精细分析
- 现场异常没有标准码，永远只能靠口头解释

## 默认输出抓手
遇到生产执行问题，优先判断：
1. 是工单机制问题，还是现场执行问题
2. 是齐套失效，还是委外协同失效
3. 是报工不真实，还是状态回写断裂
4. 一期先控关键工单，还是先控委外闭环
