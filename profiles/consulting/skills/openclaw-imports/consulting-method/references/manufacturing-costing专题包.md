# 制造业专题包：制造成本 / 料工费 / 差异分析

## 适用场景
适用于以下问题：
- 财务能出成本，但业务不信
- 毛利波动大，找不到真正原因
- 料工费归集不清，成本分析滞后
- 标准成本和实际成本长期背离
- 制造现场和财务口径长期对不上

## 核心判断
制造成本问题通常不是“成本模块没上好”，而是 **BOM、工艺、报工、入库、损耗、返工、委外、分摊规则** 一起失真。

## 常见痛点
1. 单位成本算得出来，但解释不清
2. 料、工、费差异看得到，责任落不下去
3. 返工、报废、损耗、委外差异没有被真实反映
4. 月末成本关账压力大，经营分析严重滞后
5. 产品、订单、车间、产线维度利润看不清

## 根因判断
### 业务层
- 经营上想看产品利润、订单利润、客户利润，但基础归集链条没打通

### 流程层
- 领料、报工、入库、退料、返工、报废、盘点、委外结算不规范
- 成本异常没有标准归因机制

### 系统层
- ERP 成本核算主线与 MES/WMS 执行数据断裂
- 标准成本维护机制缺失
- 实际成本计算依赖月末补录与手工调整

### 数据层
- BOM、工艺路线、工时、损耗率、产量、完工率数据不准
- 费用分摊规则不清，导致成本结果不稳定

## 方案框架
### 1. 明确成本分析层级
至少分清：
- 标准成本
- 实际成本
- 成本差异
- 产品/订单/车间/产线维度分析

### 2. 打通料工费归集链路
- 料：领退补、替代料、损耗、报废
- 工：工时、工序、设备、人工归集
- 费：制造费用、能源、折旧、外协分摊

### 3. 建立差异分析机制
重点看：
- 价格差异
- 用量差异
- 效率差异
- 产量差异
- 委外差异
- 报废/返工差异

### 4. 建立月结与经营分析双机制
- 月结满足财务核算
- 日/周经营分析服务业务动作
- 不能所有成本问题都等月末才知道

## 系统边界建议
- ERP：标准成本、实际成本、总账、存货核算、委外结算
- MES：工时、产量、工序、停机、异常、现场执行数据
- WMS：收发、退料、盘点、批次、库位
- BI：成本差异钻取与经营分析展示

## 一期建议
1. 先选核心产品线梳理料工费归集链路
2. 先治理 BOM / 工艺 / 报工 / 入库关键数据
3. 先建立标准成本维护机制
4. 先做 3~5 类核心差异分析模型
5. 先建立财务+制造联合复盘机制

## 高风险点
- 只看最终单位成本，不看差异来源
- 成本核算和制造现场脱节
- 没有标准成本维护机制，却长期解释实际波动
- 月末手工调账过多，经营分析没有行动价值

## 默认输出抓手
遇到制造成本问题，优先判断：
1. 是归集链条问题，还是核算方法问题
2. 是 BOM/报工/入库失真，还是分摊规则失真
3. 用户要的是财务关账，还是经营改善
4. 一期是先治关键数据，还是先治差异分析模型
