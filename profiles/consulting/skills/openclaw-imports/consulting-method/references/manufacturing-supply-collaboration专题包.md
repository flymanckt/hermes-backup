# 制造业专题包：供应协同 / SRM / 来料质量 / 风险预警

## 适用场景
适用于以下问题：
- 供应商交付不稳定
- 缺料频繁，计划失真
- 来料质量反复波动
- 采购和计划、质量协同差
- 供应风险总是事后才知道

## 核心判断
供应协同问题通常不是采购跟催不够，而是 **计划共享不足、供应商分层不清、来料质量闭环弱、风险预警机制缺失**。

## 常见痛点
1. 采购只看下单，不看真实交付能力
2. 计划变化没有及时同步给供应商
3. 供应商绩效只有价格，没有交付和质量维度
4. 来料异常反复，但整改闭环弱
5. 关键物料缺料总是临时救火

## 方案框架
### 1. 建供应商分层与协同策略
### 2. 建关键物料的预测/排产共享机制
### 3. 建来料质量与整改闭环
### 4. 建交付风险预警
### 5. 建采购、计划、质量联合复盘机制

## 系统边界建议
- SRM：供应商协同、交付确认、对账、整改
- ERP：采购订单、收货、库存、付款主线
- QMS：来料质量、整改、质量评分
- BI：供应风险、交付达成、质量趋势分析

## 一期建议
1. 先识别关键供应商和关键物料
2. 先做交付+质量双维度评分
3. 先打通计划变更同步机制
4. 先做高风险供应商整改闭环

## 高风险点
- 只做价格管理，不做交付和质量治理
- 计划变化不透明，供应商永远被动
- 缺少关键物料预警机制
