# 供应链与采购域专题包

## 目标
用于帮助 consulting agent 面对采购、供应协同、P2P、SRM、供应商管理、对账付款、采购透明度等问题时，能给出端到端顾问判断，而不是停留在单个 SRM 功能建议。

## 一、默认结论框架
当用户提到以下问题时：
- 请购到下单周期太长
- 采购执行不透明
- 供应商协同弱
- 对账付款慢
- OA/SRM/ERP 多头流转
- 采购价格、条款、质量、交付难统一管理
- 境外采购和移动场景支持差

优先默认判断：
1. 问题核心通常不是“采购模块不够强”，而是请购—寻源—下单—收货—对账—付款没有闭环。
2. 采购域若只优化前台审批，不优化供应商协同、对账机制、付款前置控制，收益会很有限。
3. SRM / ERP / OA / 资金系统的边界不清，是采购域长期反复返工的高频根因。

## 二、采购域核心链路
默认按这条链路分析：
1. 采购需求提出 / 请购
2. 预算与审批
3. 寻源 / 询比价 / 定标
4. 合同与条款确认
5. 采购订单下达
6. 供应商协同确认
7. 到货 / 收货 / 验收 / 质检
8. 对账 / 发票 / 应付
9. 付款执行 / 状态回写
10. 供应商绩效与持续优化

判断原则：
- 采购不是“下单功能”，而是成本、交付、质量、现金流共同作用的管理链路。
- 付款控制真正有效，必须前置到合同、验收、对账，而不是等财务付款时兜底。
- 供应商管理不能只停在档案维护，要贯穿准入、协同、绩效、异常闭环。

## 三、常见问题与根因模型

### 1. 请购到下单太慢
常见表现：
- 多层审批
- 采购员反复催确认
- 需求提了很久才落单

常见根因：
- 请购入口不统一
- 审批规则过多且重复
- 询比价和定价流程不清
- 采购权限与额度规则不清
- OA、SRM、ERP 间来回切换

顾问判断：
- 这类问题大多不是单纯审批效率问题，而是流程入口与边界设计问题。

### 2. 供应商协同弱
常见表现：
- 订单靠邮件/微信确认
- 交期变更难跟踪
- 异常靠人工追
- 供应商绩效无法量化

常见根因：
- SRM 只做了静态档案，没真正做协同
- 订单确认、交期反馈、异常反馈未在线化
- 供应商分级分类管理缺失
- 没有统一绩效指标和异常闭环

顾问判断：
- 没有供应商协同，就谈不上采购透明度和预测性管理。

### 3. 对账与付款慢
常见表现：
- 到货了不能及时对账
- 发票、收货、合同对不上
- 财务付款前临时找业务确认

常见根因：
- 三单匹配规则不清或未系统化
- 验收、质检、收货状态未回写
- 合同付款条件未结构化
- 供应商对账协同缺失
- 付款控制点后置

顾问判断：
- 对账慢、付款慢的本质通常是前链条数据和控制点没有打通。

### 4. 采购管理看不清
常见表现：
- 看不到采购执行进度
- 看不到异常批次与交付风险
- 看不到供应商质量与交付表现
- 管理层只在月底或出问题时才看到结果

常见根因：
- 过程数据未在线化
- 数据分散在 OA、SRM、ERP、Excel
- 供应商、订单、收货、发票、付款未形成统一视图

顾问判断：
- 报表缺失只是表象，实质还是交易链路和事件回写不完整。

## 四、默认诊断维度
每次至少看 6 维：

### 1. 流程机制
- 请购入口在哪
- 审批谁定规则
- 询比价和定标怎么走
- 紧急采购与例外采购如何管

### 2. 供应商治理
- 谁负责准入
- 分类分级是否存在
- 交期/质量/价格绩效是否量化
- 异常是否闭环

### 3. 交易与结算
- 订单主记录在哪
- 收货/验收/质检/对账状态是否打通
- 发票与应付链路是否标准化

### 4. 系统边界
- OA 是否侵入核心交易
- SRM 是否真正负责外部协同
- ERP 是否稳住订单/收货/应付主账
- 资金系统是否负责付款执行与状态回写

### 5. 数据与分析
- 供应商、合同、物料、价格口径是否统一
- 采购执行、对账、付款是否可视
- 异常是否能追踪到源单据

### 6. 治理责任
- 谁对交付负责
- 谁对采购周期负责
- 谁对付款时效负责
- 谁对供应商绩效负责

## 五、系统边界建议

### OA
应负责：
- 通用审批、行政协同、轻量化非核心流程

不应负责：
- 长期承载采购主交易与关键业务状态

### SRM
应负责：
- 供应商准入、协同、寻源、询比价、订单协同确认、对账协同、异常反馈

### ERP
应负责：
- 采购订单、收货、入库、应付、财务核算
- 与库存、成本、财务闭环衔接

### 资金系统
应负责：
- 付款执行、银企直联、支付状态回写、头寸与资金计划联动

## 六、关键指标抓手
建议至少看：
- 请购到下单周期
- 订单确认时效
- 到货准时率
- 对账周期
- 付款及时率
- 异常订单占比
- 供应商准时交付率
- 供应商质量合格率
- 采购降本达成率（如适用）

## 七、一期/二期建议

### 一期优先做
1. 明确 OA/SRM/ERP/资金系统边界
2. 打通请购—订单—收货—对账—付款基础闭环
3. 启用关键供应商协同场景
4. 前置付款条件与异常控制
5. 建立采购执行与异常可视化

### 二期再做
1. 供应商绩效精细化
2. 采购经营分析与预警
3. 境外采购与移动场景优化
4. 与市场行情、预测、智能推荐联动

## 八、默认追问清单
1. 你当前最痛的是审批慢、协同弱、对账慢，还是付款控制弱？
2. 现在哪些系统在管请购、订单、收货、对账、付款？
3. 供应商协同主要靠什么：系统、邮件还是微信群？
4. 财务付款前最常卡在哪一步？
5. 采购管理层最想看清什么：价格、交期、质量、异常还是付款进度？

## 九、默认输出结构
1. 结论
2. 当前问题（流程 / 协同 / 结算 / 可视化）
3. 根因判断
4. 系统边界建议
5. 一期切入点
6. 指标建议
7. 风险与前提

## 十、不要犯的错误
- 不要把采购问题简单当审批流问题。
- 不要让 OA 长期侵入核心采购交易。
- 不要只做供应商门户展示，不做交易与异常闭环。
- 不要把付款风险留到财务最后一关才处理。
- 不要只做采购报表，不做链路标准化与状态回写。
