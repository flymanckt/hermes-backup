# 采购与供应协同专题包

## 1. 结论
当前采购域的核心问题，不是单个系统功能弱，而是 **请购—寻源—下单—收货—对账—付款** 这一整条链路没有真正闭环，导致：
- 协同效率低
- 数据割裂
- 付款控制后置
- 供应商协同弱
- 管理层难以及时看清采购执行与风险

采购域应该被当成一个 **端到端治理项目**，而不是把 SRM、SAP、OA 各自单独优化。

## 2. 当前典型问题结构
结合现有需求，采购域至少存在 5 类问题：

### 2.1 流程断裂
- 请购、审批、执行脱节
- OA / SRM / SAP 多头流转
- 境外采购与移动场景支持不足

### 2.2 系统边界不清
- OA 承接了过多业务审批
- SRM 供应商管理能力未真正启用
- SAP 承担核算主系统，但前台协同不足

### 2.3 数据链路不完整
- 外采原棉批次、公检、合同、付款、到货、厂检、发票没有完整串联
- 经营分析需要内外部数据交叉验证，但数据源分散

### 2.4 控制点后置
- 付款条件与合同条款、收货验收、对账规则没有充分前置
- 风险往往在付款或对账环节才暴露

### 2.5 管理透明度不足
- 采购执行、供应商绩效、异常批次、付款进度、境外采购执行等缺少统一可视化

## 3. 目标蓝图
采购域目标不是“把 SRM 功能补齐”，而是形成以下闭环：

### 业务闭环
请购 → 寻源/询比价 → 定价与审批 → 采购订单 → 供应商协同 → 收货/验收 → 对账 → 发票 → 付款

### 数据闭环
供应商 → 合同 → 订单 → 批次 → 收货 → 质检 → 发票 → 应付 → 付款 → 分析

### 管控闭环
准入控制 → 价格与条款控制 → 三单匹配/对账控制 → 付款条件控制 → 供应商绩效管理

## 4. 推荐系统分工
### OA
负责通用审批、行政协同、非核心业务协同。
不建议继续承接核心采购主流程。

### SRM
负责：
- 供应商准入与主档协同
- 寻源、询比价、供应商协同
- 订单确认、对账协同、异常反馈

### SAP / ERP
负责：
- 采购订单主记录
- 收货、入库、应付、财务核算
- 与库存、财务、付款链路衔接

### 资金系统
负责：
- 付款执行
- 银企直联
- 付款状态回写
- 资金计划联动

## 5. 一期优先动作
### 动作 1：明确 SRM / SAP / OA 边界
必须先把以下问题定死：
- 请购入口在哪
- 采购订单主记录在哪
- 供应商协同在哪
- 通用审批留给谁
- 付款执行归谁

### 动作 2：打通采购主链路
至少打通：
请购 / 审批 / 订单 / 收货 / 对账 / 应付 / 付款

### 动作 3：补齐供应商管理能力
优先启用：
- 供应商准入
- 供应商分类分级
- 供应商协同反馈
- 供应商绩效与异常记录

### 动作 4：建立外采原棉批次链路
把公检、合同、批次、到货、厂检、发票、付款串起来，形成采购质量和结算追溯链。

### 动作 5：前置付款控制
将付款条件、到货状态、验收状态、对账状态尽量前置联动，不要等到财务付款时才发现异常。

## 6. 二期建议
- 采购经营分析与异常预警
- 内外部市场信息抓取与采购决策支持
- 境外采购与移动审批体验优化
- 电子印章与合同数字化联动

## 7. 指标建议
采购域后续应重点跟踪：
- 请购到下单周期
- 下单到到货周期
- 对账周期
- 付款及时率
- 异常批次占比
- 供应商准时交付率
- 供应商质量合格率
- 境外采购流程时效

## 8. 实施风险
- 采购、财务、业务、质量、物流各管一段，无统一 owner
- SRM 上线只做前台展示，不做流程闭环
- OA 继续侵入核心交易链路
- 供应商主数据和合同条款未统一
- 一期同时想做太多场景

## 9. 顾问建议
采购域要按“主流程闭环项目”来抓，而不是按“SRM优化需求单”来抓。真正的优先级不是功能多少，而是能否先打通采购执行与付款控制这条主线。
