# 制造业专题包：质量管理 / IQC / IPQC / OQC / CAPA

## 适用场景
适用于以下问题：
- 来料、制程、出货质量问题反复出现
- 返工返修多，但根因不清
- 质量数据很多，但对生产和采购没有形成约束
- 客诉频繁，质量改进总停留在表面
- QMS、ERP、MES 数据断裂，质量闭环跑不起来

## 核心判断
制造业质量问题通常不是“检验不够严”这么简单，而是 **标准不清、检验点设计失效、异常闭环不足、质量数据没有进入业务决策**。

## 常见痛点
1. IQC 只是拦截，不推动供应商改善
2. IPQC 发现问题，但无法快速回到工艺、设备、人员、物料根因
3. OQC 拦下问题时，损失已经发生
4. 客诉和内部不良没有打通分析
5. 返工、报废、让步接收缺少统一规则
6. CAPA 流于表面，纠正动作有，预防动作弱

## 根因判断
### 一、业务/管理层
- 质量被当成质检部门的事，不是经营指标和制造协同指标
- 质量目标没有和交付、成本、客户满意度联动

### 二、流程层
- IQC / IPQC / OQC 检验点设计不合理
- 异常升级、停线、放行、让步接收、返工判定没有标准闭环
- CAPA 没有形成跨部门动作机制

### 三、系统层
- ERP、MES、QMS、WMS、SRM 之间边界不清
- 来料不良、过程不良、客诉、供应商质量数据没有统一主线
- 质量状态没有影响采购、生产、发货等业务动作

### 四、数据层
- 质量标准、检验方案、缺陷代码、原因代码、责任代码不统一
- 不良、返工、报废、让步接收数据无法穿透分析

## 方案框架
### 1. 质量管理分四层
- **IQC**：来料质量控制，联动供应商准入、检验、退货、整改
- **IPQC**：过程质量控制，联动工艺、设备、工单、报工、异常停线
- **OQC**：出货质量控制，联动成品放行、客户要求、包装与交付
- **CAPA**：问题整改与预防，联动责任闭环与复发控制

### 2. 质量不是“发现问题”，而是“控制问题前移”
优先前移控制点：
- 供应商准入与来料标准
- 首件检验
- 关键工序控制
- 批次追溯
- 出货前风险校验

### 3. 统一异常闭环机制
至少统一：
- 不良分类
- 原因代码
- 责任归属
- 停线/放行规则
- 返工/返修/报废规则
- CAPA 时限与验证规则

### 4. 让质量数据进入经营动作
质量数据至少要驱动：
- 供应商分级与淘汰
- 工艺优化
- 设备维护
- 人员培训
- 成本差异分析
- 客诉预警与改进

## 系统边界建议
### ERP
负责：
- 物料、批次、库存状态、采购收货、成品放行关联的业务主线
- 报废、返工、存货影响、成本影响

### MES
负责：
- 工序质量、首件、巡检、过程异常、停线与现场反馈

### QMS
负责：
- 质量标准、检验方案、缺陷库、CAPA、审核、投诉闭环

### SRM
负责：
- 供应商质量协同、整改、评分、准入与淘汰联动

### WMS
负责：
- 批次、库位、冻结、隔离、待检、让步接收等库存状态执行

## 一期建议
1. 先统一缺陷代码、原因代码、责任代码
2. 先梳理 IQC / IPQC / OQC 核心检验点
3. 先建立不良-返工-报废-CAPA 的标准闭环
4. 先打通质量状态对采购、生产、库存、发货的影响
5. 先抓 1~2 类高损失质量问题做专题治理

## 二期建议
1. 推进供应商质量评分与整改闭环
2. 推进批次追溯与客户投诉闭环
3. 推进质量成本分析（预防/鉴定/内部损失/外部损失）
4. 推进质量与设备、工艺、培训联动优化

## 高风险点
- 只有检验，没有 CAPA
- 质量数据独立存在，不能影响业务动作
- 不良代码混乱，后续分析全部失真
- QMS 建了，但和 ERP / MES / SRM 没打通
- 把客诉当售后问题，不回到制造和供应链根因

## 默认输出抓手
遇到质量问题，优先判断：
1. 问题主要发生在来料、制程，还是出货后
2. 是标准缺失，还是执行失控
3. 是异常没有闭环，还是数据没有进入管理动作
4. 一期先治检验点，还是先治异常闭环和代码体系
