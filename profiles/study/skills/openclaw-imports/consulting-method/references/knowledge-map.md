# Consulting 知识库地图

## 目标
这不是资料目录，而是 consulting agent 的默认导航图。面对问题时，优先按“问题类型 → 参考材料 → 输出模板”快速进入正确路径。

## 一、按问题类型导航

### 1. 业务诊断类
适用：流程卡点、职责不清、效率低、管控弱、跨部门协同差。
优先阅读：
- `references/consulting-core-method.md`
- `references/discovery-questionnaire.md`
- `templates/business-diagnosis-template.md`
- `references/maturity-assessment-framework.md`

### 2. 系统边界类
适用：ERP / SRM / CRM / OA / 费控 / 资金系统怎么分工，哪个系统该做什么。
优先阅读：
- `references/system-boundary-handbook.md`
- `references/integration-conflict-checklist.md`
- `references/system-selection-checklist.md`
- `templates/solution-blueprint-template.md`

### 3. 数据 / BI / 主数据类
适用：报表自动化、BI 驾驶舱、主数据治理、口径不一致、AI 分析。
优先阅读：
- `references/data-governance-and-bi-framework.md`
- `references/finance-masterdata-bi专题包.md`
- `references/maturity-assessment-framework.md`
- `templates/business-diagnosis-template.md`
- `templates/solution-blueprint-template.md`

### 4. 分期实施 / 路线图类
适用：一期二期怎么切、先做什么、推进风险、项目治理。
优先阅读：
- `references/build-roadmap-handbook.md`
- `templates/delivery-roadmap-template.md`
- `templates/topic-package-output-template.md`
- `references/consulting-core-method.md`

### 5. 选型评估类
适用：先上 ERP 还是 SRM/CRM/费控/资金系统，或国产软件怎么选。
优先阅读：
- `references/system-selection-checklist.md`
- `references/build-roadmap-handbook.md`
- `templates/vendor-selection-output-template.md`

### 6. 需求池 / 项目组合管理类
适用：大量需求分诊、年度项目池排序、数字化蓝图优先级规划。
优先阅读：
- `references/enterprise-demand-portfolio-method.md`
- `references/demand-pool-master-diagnosis专题包.md`
- `cases/demand-pool-diagnosis-2026-q1.md`
- `templates/standard-consulting-deliverables-template.md`

## 四、默认工作顺序
1. 先读 `MEMORY.md`
2. 再按问题类型进入对应方法文档
3. 然后按业务域补案例/行业视角
4. 最后套合适模板输出

## 五、使用原则
- 先判断问题类型，再找材料，不要全文乱翻。
- 优先使用“方法文档 + 案例包 + 模板”的组合。
- 用户给了很多原始材料时，先提炼问题结构，不直接复述原文。
