# 系统边界判断手册

## 目标
用于判断 ERP、SRM、CRM、OA、费控、资金系统之间的职责边界，避免系统重叠建设、流程断裂和主数据冲突。

## ERP
负责核心交易、库存、订单、应收应付、总账、法定财务核算。
不适合承载所有外部协同和前台体验。

## SRM
负责供应商准入、寻源、询比价、供应商协同、对账协同。
不替代 ERP 的采购核算地位。

## CRM
负责客户、线索、商机、报价、销售过程管理。
不替代 ERP 的订单履约与财务应收。

## OA
负责通用协同审批、行政办公、人事流程。
不应成为核心交易流程主系统。

## 费控
负责费用申请、报销、预算联动、票据校验、费用前端控制。
不替代 ERP 总账和会计核算。

## 资金系统
负责银企直联、支付执行、账户与头寸管理、回款识别。
不替代 ERP 财务核算和费控前台申请。

## 三条判断原则
1. 核心交易与核算 → ERP
2. 跨组织协同体验 → SRM / CRM
3. 通用审批协同 → OA
4. 费用前台控制 → 费控
5. 支付执行与资金调度 → 资金系统
