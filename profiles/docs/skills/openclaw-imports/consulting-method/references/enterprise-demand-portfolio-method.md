# 企业需求池组合管理方法

## 目标
帮助 consulting agent 面对大批量需求时，不只做优先级排序，而是形成“组合管理”视角：哪些是保运营，哪些是补闭环，哪些是提效率，哪些是做能力建设。

## 一、需求池不是待办列表，而是投资组合
企业需求池里，需求价值和性质不同，不能放在同一优先级框架里硬排。
默认拆成四类组合：

### 1. 稳定运行组合
目的：保业务连续性、保数据准确、保关键流程不掉链子。
典型：系统缺陷修复、接口异常、性能问题、设备与网络问题。

### 2. 主流程闭环组合
目的：打通采购、销售、费用、生产、库存、资金等主链路。
典型：SRM-SAP-OA闭环、WMS-SAP打通、CRM到订单回款打通。

### 3. 管理提效组合
目的：提升报表效率、经营透明度、日常协同效率。
典型：日报自动化、BI看板、线上申请、提醒机制、电子印章。

### 4. 能力建设组合
目的：建立中长期竞争力和创新能力。
典型：AI情报、预测模型、OCR智能处理、3D展示、路线优化。

## 二、组合管理的判断原则
### 原则 1：先保运行，再谈创新
如果底层运行不稳，AI/BI/创新项目会被反复打断。

### 原则 2：先闭环，再提效
主流程没闭环时，自动化和分析的收益会打折。

### 原则 3：先数据基础，再智能化
源数据、主数据、口径不稳时，AI 只会更快地产生噪音。

### 原则 4：先少数关键项目，再全面铺开
需求池大时，宁可抓住 3-5 个高价值项目，也不要平均用力。

## 三、典型排序逻辑
常见推荐排序：
1. 稳定运行组合中的 P1
2. 主流程闭环组合中的 P1
3. 与财务/经营透明度直接相关的管理提效项目
4. 需要外部资源且依赖基础条件的能力建设项目

## 四、输出时的建议表达
默认给出：
1. 需求池总体结构判断
2. 四类组合分布
3. 当前建议优先投资的组合
4. 不建议当前优先推进的组合
5. 一期项目清单与理由
