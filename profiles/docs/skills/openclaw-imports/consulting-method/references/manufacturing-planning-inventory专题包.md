# 制造与计划库存专题包

## 目标
用于帮助 consulting agent 面对制造企业中“计划不准、库存偏高、缺料频发、生产协同混乱、成本不真”的问题时，能快速给出像顾问一样的判断，而不是泛泛谈 ERP/MES/WMS。

## 一、先给结论的默认判断框架
当用户提到以下问题时：
- 计划总在变
- 采购总是缺料或催料
- 生产排程很乱
- 库存很多但还是缺货
- 呆滞库存高
- 账实不符
- 生产成本不准
- 订单交付不稳定

优先默认判断：
1. 这通常不是单点系统问题，而是“计划-采购-生产-仓储-数据”链路没有闭环。
2. 真正根因多数不在“没有系统”，而在于：基础主数据不稳、计划机制不清、执行反馈不及时、库存口径失真、例外流程过多。
3. 如果只上 APS / MES / WMS 而不先梳理计划逻辑、物料/BOM/工艺、库存准确性和责任机制，大概率只是把混乱数字化。

## 二、制造与计划库存的核心链路
默认按这条链路看：
1. 销售预测 / 客户订单
2. 主生产计划（MPS）
3. 物料需求计划（MRP）
4. 采购计划 / 委外计划
5. 生产工单 / 排产执行
6. 领料 / 补料 / 完工 / 入库
7. 库存准确性 / 库龄 / 周转
8. 成本归集 / 差异分析

判断原则：
- 计划不是一个表，而是一套节奏 + 规则 + 责任机制。
- 库存不是结果报表，而是计划、采购、生产、仓储共同作用的结果。
- 成本不准时，往往先别怪财务，先看 BOM、工艺、领退补料、完工和盘点是否可信。

## 三、常见问题与根因模型

### 1. 计划频繁变更
常见表现：
- 日计划总改
- 采购临时插单
- 车间总说排不过来
- 销售承诺和生产能力脱节

常见根因：
- 销售预测机制缺失或随意
- 没有明确冻结区 / 滚动计划周期
- 产能约束未纳入计划
- BOM、提前期、最小起订量、损耗率等参数不准
- 紧急插单缺少审批和优先级规则

顾问判断：
- 如果计划变更无规则，先治计划治理，不要先谈高级排程。
- 如果基础参数不准，MRP 跑得再快也只是更快地产生错误建议。

### 2. 库存高但仍缺料
常见表现：
- 总库存金额高
- 现场却经常断料停线
- 常用料缺，慢料堆积
- 安全库存一刀切

常见根因：
- 物料分类策略缺失（A/B/C、关键件/长周期件）
- 替代料、共用料、专用料没区分
- 库存可用量、在途量、冻结量定义不统一
- 仓储账实不一致
- 采购批量策略与需求节奏不匹配
- 计划口径按月，执行口径按天，节奏错位

顾问判断：
- “库存高还缺料”本质是结构错配，不是简单补货不足。
- 必须先拆：总量问题、结构问题、准确性问题、策略问题。

### 3. 生产与仓储协同差
常见表现：
- 工单下了但领料慢
- 补料靠电话/微信群
- 完工入库延迟
- 在制品状态不清

常见根因：
- 工单、领料、完工节点未标准化
- 仓库按静态出入库管理，未围绕生产节拍服务
- 现场报工不及时
- 退料/补料/超耗流程失控
- WIP 在制状态不可见

顾问判断：
- 这类问题常被误以为是仓库执行问题，实际往往是工单管理和现场反馈机制缺失。

### 4. 成本不准
常见表现：
- 标准成本和实际成本偏差大
- 月末结成本很慢
- 订单毛利不可信
- 财务和业务互相不认数

常见根因：
- BOM / 工艺路线长期不更新
- 领退补料不规范
- 委外、返工、报废未完整归集
- 完工与入库时点混乱
- 库存账实不准导致结存失真

顾问判断：
- 成本问题通常是流程与数据问题的结果，不应只从核算方法层面解释。

## 四、默认诊断维度
每次都至少看 6 维：

### 1. 计划机制
- 预测、接单、MPS、MRP、排产分别谁负责？
- 冻结区和滚动周期有没有？
- 插单规则是谁拍板？

### 2. 基础数据
- 物料主数据是否稳定？
- BOM 是否准确、版本是否受控？
- 工艺路线、工时、损耗率、提前期是否可信？

### 3. 执行反馈
- 报工、领料、补料、退料、完工是否实时？
- 在制、在途、待检、冻结库存是否清楚？

### 4. 库存控制
- 安全库存逻辑是否分类管理？
- 仓库账实准确率如何？
- 库龄和呆滞分析是否进入经营动作？

### 5. 系统边界
- ERP 是否承载计划、工单、库存和成本主交易？
- MES 是否只补现场执行，不抢 ERP 主账？
- WMS 是否服务仓储执行，不另立库存真相？
- APS 是否建立在可信参数和治理机制之上？

### 6. 治理责任
- 谁对交付负责？
- 谁对库存周转负责？
- 谁对计划达成率负责？
- 谁对 BOM / 工艺 / 参数维护负责？

## 五、系统边界建议

### ERP
应负责：
- 计划主逻辑（MPS/MRP 基础能力）
- 采购、工单、库存、委外、成本主交易
- 主数据与财务核算衔接

### MES
应负责：
- 车间执行、报工、工序进度、质量采集、设备联动
- 补充生产现场透明度

不应负责：
- 长期替代 ERP 做库存主账或成本主账

### WMS
应负责：
- 上架、拣配、库位、条码、批次、盘点、出入库执行效率

不应负责：
- 成为计划口径或财务库存真相来源

### APS
适合在以下前提下考虑：
- 产能约束复杂
- 多产线多工序排产难
- 基础参数较稳定
- 企业愿意遵守计划纪律

不满足以上前提时，不要把 APS 当救命药。

## 六、关键指标抓手
建议至少看：
- 计划达成率
- 订单准交率
- 缺料停线次数 / 时长
- 库存周转天数
- 呆滞库存占比
- 账实准确率
- BOM 准确率
- 报工及时率
- 工单按期完工率
- 标准成本与实际成本偏差率

## 七、一期/二期建议

### 一期优先做
适合优先落地：
1. 统一计划节奏与责任机制
2. 清理关键物料/BOM/提前期等基础数据
3. 打通工单-领料-完工-入库基础闭环
4. 提升仓库账实准确率
5. 建立缺料、呆滞、计划达成可视化看板

### 二期再做
1. 高级排程优化
2. 深度 MES / WMS 精细化场景
3. 成本精益分析与多维模拟
4. 预测算法 / AI 计划辅助

原则：
- 没有基础数据和纪律前，不要急着上复杂优化系统。
- 先把“真实库存、真实需求、真实执行反馈”做出来，再谈高级能力。

## 八、默认追问清单
信息不足时，优先追问：
1. 你最想解决的是交付、库存、缺料，还是成本？
2. 当前计划是谁在做？按周、按日还是按订单滚动？
3. 现在用什么系统管计划、工单、库存、报工？
4. BOM、工艺、提前期、库存准确率现在大概什么水平？
5. 现场最常见的例外情况是什么：插单、缺料、返工还是账实不符？

## 九、默认输出结构
制造与计划库存场景，默认输出尽量包括：
1. 结论
2. 当前问题（计划 / 库存 / 执行 / 成本）
3. 根因判断
4. 系统与流程边界建议
5. 一期切入点
6. 关键指标
7. 风险与前提

## 十、不要犯的错误
- 不要把问题简单归因于“系统不够先进”。
- 不要在没确认基础数据质量前就推荐 APS/MES/WMS 全套上马。
- 不要只看库存总量，不拆结构、准确性和策略。
- 不要把成本失真只当财务核算问题。
- 不要忽略组织责任：计划、采购、生产、仓库、工艺、财务必须共同闭环。
