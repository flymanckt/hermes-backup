# 海外业务 / 多法人 / 跨区域协同

## 适用场景
适用于有以下特征的企业：
- 多法人、多国家/地区运营
- 海外销售、采购、仓储、服务交付并存
- 集团总部希望统一管控，但区域公司需要经营灵活性
- 涉及本地税务、币种、语言、会计准则、清结算差异

## 核心矛盾
这类问题通常不是“再上一个系统”就能解决，而是 **集团统一与区域自治、全球标准与本地合规、经营可视与执行灵活** 之间的平衡问题。

## 常见痛点
1. 总部想统一流程，但海外区域认为不适配本地业务
2. 多法人客户、供应商、物料、价格、科目体系不统一
3. 订单、发货、开票、回款、结算跨区域后责任断裂
4. 汇率、税制、转移定价、内部交易处理复杂
5. 全球报表能汇总，但无法真正穿透到经营动作
6. 不同区域系统各自为政，接口与口径长期失控
7. 海外业务推进慢，不是因为系统缺，而是本地治理与主数据先天不足

## 根因判断
### 一、管理层
- 全球化经营模式没有分层：哪些必须统一，哪些允许本地化，没有明确原则
- 总部常把“标准化”理解成“所有区域一个流程”，导致执行反弹

### 二、流程层
- Lead-to-Cash、Order-to-Cash、Procure-to-Pay 在不同区域没有统一主线定义
- 内部交易、跨法人结算、跨境采购/销售没有清晰责任链

### 三、系统层
- CRM、ERP、本地税务系统、WMS、资金系统、报关/物流系统之间边界混乱
- 有的区域用集团 ERP，有的区域用本地系统，缺少全球状态总线和统一主数据映射

### 四、数据层
- 全球客户、供应商、物料、组织、科目、项目、银行账户主数据没有统一治理
- 法人口径、管理口径、区域口径、全球口径混在一起，报表看起来统一，实际上不可追溯

## 判断原则
### 1. 全球统一的，不等于全球完全一致
必须先分成三层：
- **集团统一层**：主数据标准、科目框架、核心经营指标、权限原则、核心控制点
- **区域适配层**：本地税务、票据、语言、币种、会计政策映射、付款习惯
- **本地执行层**：客户协同、仓配策略、局部运营流程

### 2. 先统一骨架，再允许本地差异
不能一开始要求所有区域流程完全相同。
先统一：
- 主数据编码规则
- 核心单据主线
- 状态定义
- 指标口径
- 跨法人交易规则

### 3. 多法人问题，本质是责任链问题
重点不是系统有几个，而是以下责任谁负责：
- 客户归属
- 订单归属
- 收入归属
- 库存归属
- 开票/税务归属
- 回款认领
- 内部结算与对账

## 系统边界建议
### CRM
负责：
- 全球客户/商机/报价前端管理
- 客户分层、销售过程、机会协同
- 但不承担最终财务归属判断

### ERP
负责：
- 各法人订单、库存、应收应付、收入成本、总账核算
- 内部交易、跨法人结算、财务归集
- 是法定财务与交易主系统

### 本地税务/电子发票/合规系统
负责：
- 本地税票、申报、监管要求承接
- 不应反向替代 ERP 成为经营主系统

### 资金系统
负责：
- 多账户资金可视、跨境支付、账户管理、头寸调度、回款识别

### BI / 数据平台
负责：
- 全球口径、区域口径、法人口径分层建模
- 经营分析穿透，不直接替代源系统责任

## 典型设计抓手
### 1. 全球主数据模型
至少统一：
- 客户
- 供应商
- 物料/商品
- 组织/法人/利润中心
- 科目与管理维度
- 项目/合同（如适用）
- 银行账户

### 2. 多账簿/多口径设计
要明确区分：
- 法定报表口径
- 集团管理口径
- 区域经营口径
- 专题分析口径

### 3. 跨法人交易闭环
要先定义清楚：
- 谁对外签单
- 谁负责发货/履约
- 谁确认收入
- 谁承担库存
- 谁收款
- 内部结算如何跑

### 4. 全球 + 本地治理机制
必须建立双层治理：
- 集团架构委员会 / 主数据委员会 / 模板治理机制
- 区域 key user / 本地流程 owner / 合规 owner

## 一期建议
1. 先做全球主数据与组织架构梳理
2. 先定义全球统一的核心流程骨架与状态模型
3. 先选 1~2 个关键区域做模板试点
4. 先打通 O2C 或 P2P 一条跨区域主链路
5. 先建立全球指标口径字典和穿透报表样板

## 二期建议
1. 推进跨法人内部交易与结算标准化
2. 推进全球资金可视与跨境回款认领
3. 推进全球客户与供应商分层治理
4. 推进区域复制与模板推广

## 高风险点
- 一开始就要求全球一套流程一套系统，不考虑本地合规
- 先上 BI，看板很漂亮，但主数据和归属逻辑没统一
- 海外系统本地化严重，但总部没做映射层设计
- 多法人收入、库存、回款归属不清，导致经营分析长期失真
- 项目推进只靠 IT，不建立全球+区域联合治理机制

## 默认输出抓手
遇到海外多法人问题，优先判断：
1. 问题是全球标准缺失，还是本地适配失控
2. 主数据没统一，还是责任归属没定义
3. 是流程骨架没统一，还是接口和口径没治理
4. 哪些必须集团统一，哪些应该区域自治
5. 一期是先做模板试点，还是先做主数据和口径治理
