# 制造业专题包：设备管理 / TPM / OEE / 备件

## 适用场景
适用于以下问题：
- 设备故障频繁，计划总被打乱
- 停机多，但原因不透明
- 点检保养流于形式
- 备件库存高但关键件仍缺
- OEE 口号很多，真正改善很少

## 核心判断
设备问题通常不是“维修不够勤”，而是 **设备台账、预防性维护、故障分类、备件管理、生产协同** 没有形成闭环。

## 常见痛点
1. 故障响应靠师傅经验，缺少标准机制
2. 点检、保养、维修、备件彼此割裂
3. 设备停机影响产能，但计划侧看不见
4. OEE 数据有，但不可行动
5. 备件采购与消耗分析脱节

## 根因判断
- 台账不准：设备、部件、寿命、历史记录不完整
- 流程不闭环：点检、保养、维修、复盘脱节
- 数据不统一：停机原因、维修代码、备件代码混乱
- 系统不联动：设备状态不能反馈计划、成本、质量

## 方案框架
### 1. 建设备主数据与层级台账
至少统一：设备、产线、部件、关键件、寿命周期、保养策略

### 2. 建预防性维护机制
- 点检
- 保养
- 巡检
- 预警
- 停机升级规则

### 3. 建停机原因与维修闭环
统一：
- 故障代码
- 停机原因
- 维修动作
- 责任归属
- 复盘结论

### 4. 建备件协同机制
- ABC 分类
- 关键件安全库存
- 生命周期管理
- 备件成本分析

### 5. 让设备数据进入经营动作
设备数据要进入：
- 计划排程
- 产能评估
- 成本分析
- 质量改善

## 系统边界建议
- ERP/EAM：设备台账、备件、维修工单、资产与成本
- MES：停机、设备状态、现场采集、OEE
- WMS：备件库存、出入库、批次/库位
- BI：故障趋势、停机分析、备件消耗、OEE 看板

## 一期建议
1. 先建关键设备台账
2. 先统一停机原因和维修代码
3. 先抓关键产线做 TPM 试点
4. 先打通设备状态对计划和产能的影响
5. 先治理关键备件库存策略

## 高风险点
- 只做点检表，不做闭环
- 只看 OEE，不看故障根因
- 备件库存靠经验拍脑袋
- 设备数据和生产计划脱节
