# 案例模板：海外多法人协同诊断

## 结论
这类项目的核心，不是简单选系统，而是先把 **全球统一层 / 区域适配层 / 本地执行层** 划清，再决定系统模板怎么建、哪些主数据先统一、哪些流程先试点。

## 典型现状
- 总部希望全球可视、统一管控
- 区域公司强调本地法规、语言、币种、客户习惯差异
- 系统形态混杂：总部 ERP + 区域本地系统 + Excel + 本地税务平台
- 全球经营分析难穿透，尤其在客户、库存、回款、利润归属上失真

## 关键问题
1. 集团统一要求与区域经营现实冲突
2. 客户/供应商/物料/组织主数据不统一
3. 多法人订单、发货、收入、回款、库存归属不清
4. 全球报表汇总能做，但经营责任追不清

## 根因判断
### 管理
- 没有明确全球模板治理原则

### 流程
- 跨区域 O2C / P2P / 内部交易没有清晰主线

### 系统
- 模板系统、本地系统、税务/物流系统边界混乱

### 数据
- 全球口径与区域口径混在一起，没有映射模型

## 建议方案
### 一期
1. 建全球主数据标准
2. 建全球指标口径字典
3. 选关键区域做模板试点
4. 先打通一条跨区域主链路

### 二期
1. 推进跨法人交易标准化
2. 推进资金可视与回款认领
3. 推进区域复制与模板推广

## 风险提示
- 不能用“全球统一”压掉本地合规需求
- 不能先做看板再补主数据
- 不能忽视跨法人责任归属设计
