# 制造业组合案例包：主数据混乱 + MRP 跑偏 + 生产执行失真

## 结论
这类问题的根通常不在现场，而在 **物料/BOM/工艺/工作中心主数据治理失效**，导致计划、执行、成本、质量全部一起偏。

## 典型现状
- 系统里能跑出计划，但现场不认
- 工单下了很多，实际齐套率低
- BOM 和工艺版本经常不一致
- 报工、入库、在制数量不可信

## 问题拆解
### 主数据层
- 同物多码、BOM 失真、工艺版本混乱、工作中心能力参数不准

### 计划层
- MRP 建议依赖错误基础数据，结果天然失真

### 执行层
- 工单、报工、领退料、在制状态没有可信闭环

### 成本/质量层
- 因为主数据和执行数据失真，成本和质量分析都不可信

## 建议方案
### 一期
1. 识别关键产品线主数据问题
2. 建物料/BOM/工艺版本治理机制
3. 建工单前齐套校验
4. 建报工与入库回写规则
5. 建主数据质量指标和责任归口

### 二期
1. 推进更多产品线复制
2. 推进主数据变更审批和追溯
3. 推进计划、执行、成本、质量联动分析

## 风险提示
- 不治理主数据，却想靠系统参数救结果
- 只清历史台账，不建长期机制
- 现场数据采集弱，后续分析都会失真
