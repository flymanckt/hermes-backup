# 案例模板：制造业质量管理诊断

## 结论
质量问题如果长期重复出现，通常不是“检验不够”，而是 **质量控制点设计、异常闭环、代码体系、系统联动** 没有真正建立起来。

## 常见现状
- 来料、制程、出货、客诉各自为政
- 不良很多，但责任和根因长期模糊
- CAPA 每次都做，但复发率还是高
- 质量数据存在，但没有进入采购、生产、库存和发货控制

## 核心问题
1. IQC / IPQC / OQC 检验点不清
2. 不良/原因/责任代码混乱
3. 返工、报废、让步接收规则不统一
4. QMS、ERP、MES、SRM 没形成闭环

## 根因判断
- 标准层：检验标准和判定规则不统一
- 流程层：异常升级与 CAPA 机制失效
- 系统层：质量状态不能驱动业务动作
- 数据层：分析口径和代码体系失真

## 建议方案
### 一期
1. 统一缺陷/原因/责任代码
2. 明确 IQC / IPQC / OQC 检验点
3. 建立返工/报废/CAPA 标准闭环
4. 打通质量状态对采购、生产、库存、发货的影响

### 二期
1. 推进供应商质量管理
2. 推进批次追溯与客诉闭环
3. 推进质量成本分析

## 风险提示
- 不能只上 QMS，不改业务闭环
- 不能只有检验，没有纠正预防机制
- 不能让质量数据停留在报表层
