# 专题包输出模板

适用于 consulting agent 形成较正式的专题交付时使用。

## 1. 结论
- 一句话定义问题本质与建议方向

## 2. 专题范围
- 本次覆盖哪些业务域 / 系统 / 数据 / 组织范围
- 不覆盖什么

## 3. 现状问题结构化拆解
- 问题 1
- 问题 2
- 问题 3
- 典型表现

## 4. 根因模型
### 流程
### 系统
### 数据
### 治理

## 5. 流程方案
- 应如何改流程
- 应前置哪些控制点
- 哪些节点要标准化

## 6. 系统边界方案
- ERP / SRM / CRM / OA / 费控 / 资金系统 / MES / WMS / BI 各自职责
- 主系统与协同系统如何分工

## 7. 数据与指标方案
- 主数据归属
- 关键交易链路
- 指标抓手
- 可视化建议

## 8. 实施路线图
### 一期
- 范围
- 目标
- 成果

### 二期
- 范围
- 目标
- 成果

### 三期（如有）
- 范围
- 目标
- 成果

## 9. 风险与前提
- 组织前提
- 数据前提
- 系统前提
- 项目治理前提

## 10. 附录
- 关键清单
- 关键指标
- 角色责任
- 默认追问清单
